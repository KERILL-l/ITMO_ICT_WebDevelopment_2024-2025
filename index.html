Разработка веб-приложения RacingLeaderboard

ВВЕДЕНИЕ

Разработано веб-приложение RacingLeaderboard на Django для управления гонками и водителями.

Технологии: Django 4.x, SQLite3, Bootstrap 5.3, HTML5, CSS3

Навигационное меню с Bootstrap

Пагинацию данных

Поиск с пагинацией

==============================================================

АРХИТЕКТУРА

Паттерн MVT (Model-View-Template). URL маршрутизирует запрос на view функцию. View обрабатывает логику, получает данные из БД через ORM, передает в шаблон. Шаблон рендерит HTML с Bootstrap.

Структура:

races/ - приложение Django (models, views, forms, urls)

templates/races/ - HTML шаблоны

db.sqlite3 - база данных

manage.py - управление проектом

==============================================================

МОДЕЛИ ДАННЫХ

8 таблиц: CustomUser, Team, Driver, Car, Race, RaceResult, Comment, Registration

CustomUser - расширенный пользователь (поля: username, email, bio, avatar)

Team - команда (name, description, founded_year)

Driver - профиль водителя (experience, driver_class, description)

Car - машина водителя (make, model, year)

Race - гонка (name, location, date, description)

RaceResult - результат гонки (position, time, points)

Comment - комментарий (text, rating, created_at)

Registration - регистрация на гонку (driver, race)

==============================================================

РЕАЛИЗОВАННЫЕ КОМПОНЕНТЫ

5.1 НАВИГАЦИОННОЕ МЕНЮ

Файл: templates/races/base.html

Меню содержит логотип, пункты: Гонки, Водители, Команды.

Динамические пункты:

Если залогинен: Мой профиль, Администратор, Выход

Если не залогинен: Регистрация, Вход

Адаптивность: на ПК меню видно полностью, на мобильных сворачивается в hamburger-меню благодаря Bootstrap классам navbar-expand-lg и collapse.

5.2 ПАГИНАЦИЯ

Файл: races/views.py

Используется Django Paginator для разбиения данных:

Гонки: 5 на странице

Водители: 6 на странице

Команды: 10 на странице

View создает объект Paginator, получает нужную страницу через get_page(), передает в шаблон.

В шаблоне выводятся кнопки: Первая, Назад, номера страниц (текущая выделена), Вперед, Последняя.

5.3 ПОИСК С ПАГИНАЦИЕЙ

Файл: races/views.py и templates/

Поиск для гонок: по названию, локации, описанию Поиск для водителей: по имени, фамилии, команде, описанию

Используются Q объекты с оператором | (ИЛИ) и icontains (содержит без учета регистра).

При поиске применяется фильтр к queryset, затем к отфильтрованным результатам применяется пагинация.

При навигации по страницам параметр search сохраняется в URL (?page=2&search=Ferrari).

==============================================================

ОСНОВНЫЕ СТРАНИЦЫ

race_list.html - список гонок с поиском и пагинацией race_detail.html - информация о гонке, результаты, комментарии driver_list.html - список водителей с поиском и пагинацией driver_detail.html - профиль водителя team_list.html - список команд с пагинацией register.html - регистрация пользователя add_comment.html - добавление комментария create_driver_profile.html - создание профиля водителя edit_driver_profile.html - редактирование профиля race_leaderboard.html - таблица результатов гонки

==============================================================

VIEW ФУНКЦИИ

race_list - выводит гонки с поиском и пагинацией (5 на странице) driver_list - выводит водителей с поиском и пагинацией (6 на странице) team_list - выводит команды с пагинацией (10 на странице) race_detail - показывает детали гонки с результатами и комментариями driver_detail - показывает профиль водителя register_user - регистрирует нового пользователя create_driver_profile - создает профиль Driver (требует login) edit_driver_profile - редактирует профиль Driver (требует login, только владелец) add_comment - добавляет комментарий к гонке (требует login) register_for_race - регистрирует водителя на гонку (требует login) unregister_from_race - удаляет регистрацию (требует login) race_leaderboard - показывает результаты гонки

==============================================================

ТЕХНИЧЕСКИЕ ДЕТАЛИ

Навигационное меню использует Bootstrap классы navbar, navbar-expand-lg, navbar-toggler для адаптивности. JavaScript из Bootstrap автоматически обрабатывает collapse меню на мобильных.

Пагинация использует Django Paginator класс. View получает параметр page из GET, вызывает get_page(page_number), получает объект Page с нужными объектами.

Поиск использует Q объекты для построения сложных фильтров. Фильтр применяется до пагинации. Параметр search передается в каждую ссылку пагинации через {% if search_query %}&search={{ search_query }}{% endif %}.

Аутентификация использует встроенную Django систему. Декоратор @login_required защищает view функции. Проверка user.is_authenticated в шаблонах.
